**Key Concepts**:

1. **Layered Architecture**:
   - **Generation Layer**: Cortex auto-generates docs
   - **Storage Layer**: GitHub (version control, GitOps)
   - **Authoring Layer**: Obsidian (graph-based navigation)
   - **Presentation Layer**: Static site (MkDocs/Docusaurus/Quartz)

2. **Curation Workflow** (5 stages):
   - **Triage**: Classify and route `_inbox/` contents
   - **Enrich**: Add backlinks, tags, update index
   - **Health Check**: Find broken links, stale docs, orphans
   - **Consolidate**: Merge duplicates, archive old versions
   - **Report**: Weekly digest of vault health

3. **Trust Levels** (0-4):
   - **Level 0** (Current): Observer - read-only, reports
   - **Level 1**: Suggester - create PRs
   - **Level 2**: Minor Editor - direct commits to inbox/index
   - **Level 3**: Full Editor - direct commits except meta/
   - **Level 4**: Self-Governing - can modify own rules

4. **Design Principles**:
   - **Inbox-First**: All new content enters through `_inbox/`
   - **Suggest, Don't Act**: Cortex proposes; humans approve
   - **Earn Trust**: Write access granted incrementally
   - **Preserve History**: Never delete; archive with redirects
   - **Human-Readable**: All actions must be explainable
